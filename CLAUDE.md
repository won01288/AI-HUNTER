# 프로젝트: AI 이직 컨설턴트 서비스 (가칭)

## 개요
개인별 이직 선호도(지역/직무/경력/기업규모/희망연봉)를 프로파일링하고,
워크넷 등 공개된 채용정보를 재모델링해 정기적으로 맞춤 이직처를 추천하는
서비스. 원본 채용공고는 DB에 저장하지 않고, 요청 시점에 실시간으로 가공한
요약자료를 로그인한 개인 사용자에게만 노출하며, 항상 출처 링크로 안내한다
(콘텐츠 재배포형이 아닌 "AI 컨설턴트형" 서비스).
채용정보 수집은 원래 워크넷 오픈API 연동을 전제로 설계했으나, 현재는
오픈API 키 발급/승인이 막혀 있어 베타 기간에는 워크넷 웹사이트 크롤링으로
대신한다 (아래 "지켜야 할 규칙" 참고). 오픈API 키가 발급되면 크롤러 대신
API 호출로 전환하는 것을 우선 검토한다.

## 현재 단계
Phase 1: 온보딩(선호도 입력/프로파일링) UI + DB 스키마 구현 중.
Phase 2(채용정보 연동·매칭), Phase 3(정기 추천 발송) 순서로 구현 예정
전체 로드맵은 docs/roadmap.md 참고, 화면 상세 스펙은 docs/onboarding-flow.md 참고.

## 스택
- Frontend: Next.js (App Router)
- 스타일링: Tailwind CSS
- DB: Turso (libSQL / SQLite 호환)
- 배포: Render
- 인증: 이메일/비밀번호 + 카카오 로그인을 Phase 1에서 함께 구현.
  카카오는 `kakao_id`(카카오 고유 회원번호)를 식별자로 쓰고, 이메일
  동의항목은 카카오 "비즈 앱 전환" 심사가 끝나기 전까지 못 받을 수
  있으므로 `users.email`은 nullable로 설계한다 (db-schema.md 참고).
- 알림 발송: 카카오 알림톡 / 이메일 (Phase 3)

## 화면 흐름 순서
가입(로그인) → 온보딩. 이메일/비밀번호 또는 카카오 로그인, 둘 중 하나로
계정을 먼저 만들고 로그인한 상태에서만 온보딩 화면에 진입할 수 있다.
비회원 온보딩은 지원하지 않는다.

## 폴더 구조
- `src/app/` — Next.js 라우트 (온보딩 화면은 `src/app/onboarding/`)
- `src/lib/db.ts` — Turso 클라이언트 초기화
- `docs/` — 기획 문서. 작업 전에 관련 문서를 먼저 읽을 것.

## 지켜야 할 규칙
- 개인정보는 최소 수집 원칙. 회원 식별은 이메일/소셜 ID 수준으로 최소화한다.
  민감정보 암호화, 삭제(파기) 로직은 Phase 1에서 구현하지 않고
  `// TODO(privacy): ...` 주석으로만 표시해둔다. 실제 구현은 별도 Phase에서 진행.
- 채용정보는 원문을 통째로 저장/재게시하지 않는다. DB에는 제목, 지역, 직종,
  급여 범위, 원본 URL 정도의 메타데이터만 저장한다.
- 워크넷 채용정보는 오픈API 키 미승인 상태라 웹사이트 크롤링으로 수집한다
  (자세한 절차는 docs/roadmap_phase2.md 참고). robots.txt·이용약관 사전
  확인, 요청 간 딜레이, User-Agent 명시를 반드시 지킨다. 오픈API 키가
  발급되면 크롤러 대신 API 호출로 전환하는 것을 우선 검토한다.
- 채용공고 요약자료는 DB에 저장하지 않고, 사용자 요청 시점에 실시간으로
  가공해 로그인한 개인 사용자에게만 노출한다 (불특정 다수 대상 공개/재배포 금지).
- UI 카피에 "취업추천", "취업지원" 문구를 쓰지 않는다 (직업정보제공사업
  신고 전 규제 대응 목적). 대신 "맞춤 채용정보 안내" 같은 표현을 쓴다.
- 커밋은 기능 단위로 작게 나눈다. 한 커밋 = 한 가지 변경.
- 개발 초보자용 프로젝트이므로 복잡한 로직에는 "왜 이렇게 했는지" 주석을 남긴다.
- 새 라이브러리를 추가하기 전에 먼저 이유를 설명하고 사용자 확인을 받는다.
- 비밀번호는 반드시 해시(bcrypt 등)로 저장한다. 평문 저장 절대 금지.

- Turso 토큰, 세션 시크릿 등은 `.env.local`에만 두고 커밋하지 않는다.
  `.env.example`에 키 이름만 남겨 어떤 값이 필요한지 표시한다.
  실제 값이 든 `.env.local`, `.env`는 `.gitignore`에 반드시 포함한다.
  
- 회원가입 시 개인정보 수집·이용 동의 체크박스를 반드시 넣는다
  (본문 내용은 임시 문구로 시작해도 되지만, 체크 여부와 시각은 DB에 기록한다).
- 카카오 로그인은 "비즈 앱 전환" 심사 전까지 이메일을 못 받아올 수 있다.
  이메일이 없어도 회원가입·로그인이 끊기지 않도록 이메일에 의존하지 않는
  코드 경로(카카오는 kakao_id로 식별)를 항상 유지한다.

## 명령어
- `npm run dev` — 로컬 개발 서버
- `npm run build` — 프로덕션 빌드
- (Turso CLI 연동 후 마이그레이션 명령 추가 예정)

## 작업 방식
- 새 기능을 시작할 때는 먼저 계획(plan mode)을 세우고 승인받은 뒤 구현한다.
- 변경 후에는 diff를 사람이 직접 확인한다. 자동 커밋/푸시는 하지 않는다.
