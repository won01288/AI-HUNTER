# PRD — AI 이직 컨설턴트 서비스

## 한 줄 정의
개인의 이직 선호도를 정밀하게 프로파일링하고, 공개 채용정보를 재모델링해
"나만을 위한 이직 컨설턴트"처럼 정기적으로 맞춤 이직처를 추천하는 서비스.

## 핵심 원칙
1. 원본 채용 사이트(사람인/잡코리아/인크루트 등)의 콘텐츠를 그대로 복제해
   재게시하지 않는다. 사용자에게는 항상 원본 링크를 안내한다.
2. 무료 버전은 직업정보제공사업 신고 없이 운영 가능한 범위로 제한한다.
   유료 전환 시점에 신고 여부를 재검토한다.
3. 개인정보는 최소 수집. 정밀한 보호 조치(암호화, 파기 정책 등)는
   서비스가 유료로 확장되는 시점에 별도로 설계한다.

## 서비스 구조 (3단계)

0. **가입/로그인** — 이메일/비밀번호 또는 카카오 로그인으로 계정을 만든다.
   온보딩은 로그인한 사용자만 진행할 수 있다 (비회원 온보딩 미지원).
   카카오는 개인 개발자 계정으로 바로 연동 가능하지만, 이메일 동의항목은
   카카오 "비즈 앱 전환" 심사가 필요해 초기에는 이메일 없이 카카오
   고유 ID만으로 가입되는 사용자가 있을 수 있다.
1. **온보딩** — 로그인 직후, 사용자의 이직 선호도(지역/직무/경력/기업규모/
   희망연봉/우선순위)를 입력받아 프로파일을 만든다. → 현재 개발 단계
2. **매칭** — 워크넷 오픈API 등에서 채용정보를 가져와 사용자 프로파일과
   비교해 스코어링한다.
3. **추천 발송** — 정기적으로(예: 주 1회) 상위 매칭 공고를 이메일 또는
   카카오 알림톡으로 발송한다.

## 향후 유료 확장 (Phase 4+, 지금 설계 범위 아님)
- 자기소개서 작성 지원 (AI 첨삭)
- 나만의 AI 직무 포트폴리오 빌더
- 프리미엄 매칭 (직업정보제공사업 신고 후 기능 확대 검토)

## 지금 스프린트의 범위
Phase 1(가입+온보딩)만 구현한다. 매칭 엔진과 발송 기능은 다음 스프린트.
- 이메일/비밀번호 회원가입·로그인 + 카카오 로그인 동시 구현
- 카카오 개발자 계정 본인인증 → 비즈 앱 전환 신청(이메일 동의항목,
  승인까지 시간이 걸릴 수 있음을 감안하고 진행)
- 가입 시 개인정보 수집·이용 동의 체크 (문구는 임시, 기록은 필수)
- 3단계 온보딩 폼 (docs/onboarding-flow.md 참고)
- Turso에 사용자/선호도 데이터 저장 (docs/db-schema.md 참고)

## 성공 기준 (Phase 1)
- 신규 사용자가 이메일로 가입 → 로그인 → 필수 5문항 입력까지
  3분 이내에 완료할 수 있다.
- 온보딩 중간에 이탈해도 재로그인 시 이어서 입력할 수 있다.
- 입력된 선호도가 DB에 정확히 저장되고, 마이페이지에서 수정 가능하다.
