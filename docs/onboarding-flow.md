# 온보딩 플로우 상세 스펙

원칙: 처음엔 가볍게, 쓸수록 정교하게. 가입 직후 이탈을 막기 위해 필수
입력을 최소화하고, 이후 단계에서 점진적으로 정보를 보강한다.

전체 순서: **0단계(가입/로그인) → 1단계(필수 5문항) → 2단계(선택 정교화)**.
비회원은 온보딩에 진입할 수 없다. 로그인 직후 `user_preferences` 레코드가
없으면 자동으로 1단계로 리디렉션한다.

## 0단계 — 가입/로그인

화면: `src/app/signup/page.tsx`, `src/app/login/page.tsx`

**방식 A. 이메일/비밀번호**

| 항목 | 입력 방식 | DB 컬럼 (users) | 비고 |
|---|---|---|---|
| 이메일 | 텍스트 입력 | email | 중복 체크 |
| 비밀번호 | 텍스트 입력(마스킹) | password_hash | bcrypt 해시 저장, 평문 저장 금지 |
| 개인정보 수집·이용 동의 | 체크박스 (필수) | privacy_agreed_at | 문구는 임시 가능, 체크 시각은 반드시 기록 |

**방식 B. 카카오 로그인**

| 항목 | 처리 방식 | DB 컬럼 (users) | 비고 |
|---|---|---|---|
| 카카오 고유 회원번호 | OAuth 인가 후 자동 수집 | kakao_id | 이메일 대신 이 값으로 사용자를 식별한다 |
| 이메일 | 비즈 앱 전환 심사 통과 전에는 못 받을 수 있음 | email (nullable) | 못 받아도 가입 흐름이 끊기면 안 된다 |
| 닉네임/프로필사진 | OAuth 인가 후 자동 수집 (기본 동의항목, 심사 불필요) | name (선택 저장) | |
| 개인정보 수집·이용 동의 | 체크박스 (필수, 카카오 인가와 별개로 서비스 자체 동의) | privacy_agreed_at | |

두 방식 모두 가입 완료 → 자동 로그인 → 1단계로 이동.

## 1단계 — 필수 5문항 (로그인 직후, 목표 소요시간 1분 이내)

화면: `src/app/onboarding/step1/page.tsx`

| 항목 | 입력 방식 | DB 컬럼 (user_preferences) | 비고 |
|---|---|---|---|
| 희망 근무지역 | 시/도 단위 복수선택 칩 | regions (JSON 배열 문자열, **워크넷 지역코드값** 저장) | 자유텍스트 아님 — 나중에 매칭 시 재매핑 없도록 처음부터 코드값으로 저장 |
| 직무 대분류 | 드롭다운/검색형 선택 | job_categories (JSON 배열 문자열, **워크넷 직종코드값** 저장) | 워크넷 공통코드 API에서 코드/명칭 목록을 미리 받아 select 옵션으로 사용 |
| 총 경력 연차 | 숫자 입력 또는 슬라이더 | experience_years (정수) | 0 = 신입 |
| 희망 연봉 범위 | 최소~희망 슬라이더 | salary_min, salary_desired (정수, 만원 단위) | |
| 이직 긴급도 | 단일 선택 (즉시 / 3개월 내 / 좋은 기회면) | urgency (문자열) | |

이 5개 입력이 끝나면 `user_preferences.onboarding_completed_at`을 현재
시각으로 기록하고 대시보드로 이동시킨다. (Phase 2 전까지는 대시보드에
"매칭 준비 중" 안내만 표시)

**재방문 처리**: 로그인 후 `onboarding_completed_at`이 비어 있으면 무조건
1단계로 보낸다. 이미 입력한 항목은 폼에 채워진 상태로 보여주고 이어서
입력하게 한다 (처음부터 다시 입력하게 하지 않는다).

## 2단계 — 정교화 (선택, 첫 방문 후 노출)

화면: `src/app/onboarding/step2/page.tsx` — "더 정확한 추천을 원하시면" 배너로 유도

| 항목 | 입력 방식 | DB 컬럼 | 비고 |
|---|---|---|---|
| 기업 규모 선호 | 다중 선택 (대기업/중견/중소/스타트업) | company_size | |
| 우선순위 가중치 | 드래그 순위 또는 슬라이더 배분 (합 100) | priority_weights 테이블 (criterion, weight) | 기준: 지역/연봉/직무/기업규모 |
| 제외 조건 | 자유 태그 입력 | exclusions (JSON 배열, 선택 항목) | 특정 산업/기업 블랙리스트 |

**가중치 기본값 정책**: 사용자가 2단계를 건너뛰어 `priority_weights`에
값이 없으면, 매칭 로직(Phase 2)에서 지역/연봉/직무/기업규모를 각 25씩
균등 배분한 것으로 간주한다. DB에 기본값을 미리 넣어두지 않고,
매칭 코드에서 "레코드 없으면 25/25/25/25로 취급"하는 방식으로 처리한다.

## 3단계 — 지속 학습 (Phase 2 이후, 추천 결과 노출 시)

추천 카드에 "관심있음 / 관심없음" 버튼 → 클릭 로그를 저장해 이후
가중치 보정에 활용한다. (테이블 설계는 Phase 2에서 별도 진행)

## 유효성 검사 규칙
- salary_min ≤ salary_desired
- regions, job_categories는 최소 1개 이상 선택해야 다음 단계로 진행 가능
- experience_years는 0 이상 정수만 허용

## UI 카피 주의사항
- "취업추천", "취업지원" 표현 금지 → "맞춤 채용정보 안내"로 대체
- 진행률 표시(1/3, 2/3 등)로 이탈률을 낮춘다
